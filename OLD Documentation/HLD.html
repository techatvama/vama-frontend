Key Data Flows
Critical paths through the system
Student Portal → Browse Packages → GET /student/packages
(grade + course filter) → Confirm Modal → Razorpay Checkout → POST /verify → Invoice + StudentPackage created
Teacher marks Attendance → Attendance table → StudentPackage.sessions_used++ → Low session / expiry alert → Renewal CTA shown
Admin creates Package → POST /admin/packages → grades[] + courses[] stored as JSON → Publish toggle ON → Visible to eligible students only
Subscription nears renewal → Admin dashboard bell → Send Reminder (WhatsApp / Email) → Student sees Renew CTA → Payment → new Invoice
