Key Relationships

staff → batches (teacher_id FK)
batches → class_sessions (batch_id FK)
students → enrollments → batches (many-to-many)
class_sessions → attendance (session_id + student_id)
class_cancellations.new_session_id → class_sessions (reschedule audit)
syllabus → modules → contents (tree)
Key Data Flows
🔄 Real-Time Sync (Polling)
All three portals poll every 20s via setInterval
→ Attendance changes propagate to all portals
→ LiveBadge shows last refresh time (HH:mm)
✋ Cancel vs Reschedule
Admin cancel → session.status = 'cancelled'
Student cancel → attendance.status = 'student_cancelled'
Reschedule → ClassCancellation record + new enrollment
✅ Attendance Enforcement
Teacher must write feedback before marking Present/Absent
PUT /sessions/{id}/attendance/{sid}?require_feedback=true
Backend returns HTTP 422 if notes is empty
