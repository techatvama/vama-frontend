Multi-Center Data Isolation Principles

Single Root of Truth: CENTER is the root entity. Every other entity must trace back to a single center via foreign keys.
Direct Partition Key: center_id is present on all core entities (STAFF, STUDENT, BATCH, INVOICE, AUDIT_LOG). This enables direct, fast filtering without joins.
Transitive Closure: Secondary entities (CLASS_SESSION, ATTENDANCE, ENROLLMENT) inherit center scope via their parent relationships. Querying a CLASS_SESSION automatically scopes to its BATCH → CENTER.
Role-Based Enforcement: Database queries automatically filtered: center_admin sees only their `center_id`; super_admin bypasses filter. Enforcement at API layer via `require_roles()` + query filter.
Audit Trail Integrity: AUDIT_LOG records include `center_id`, enabling compliance reporting and cross-center analytics without data leakage.
Configuration Flexibility: APP_SETTING allows `center_id = NULL` for global settings or `center_id = N` for per-center overrides (logo, phone prefix, invoice number format).
